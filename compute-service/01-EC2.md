# Amazon Elastic Compute Cloud (EC2)
If you are interested in tech, maybe you have heard about EC2 but mostly what you heard is EC2 instance. This two kinda different because EC2 instance is one of many features in EC2. It provides on-demand, **scalable computing capacity** in AWS. With EC2 we can launch as many or as few virtual server as we want, configure network and security based on what we need, and determine the storage required.

You will hear the word scalable a lot in this repo because this is what makes cloud provider superior than the other. You can add capacity (scale up) to handle more compute-heavy task, such as spikes in website traffic, costumer increase on our system, or even data-processing that will take time to finish. You can also reduce the capacity (scale down) again when the usage decrease. You can do this with other provider, but the convenient offered by AWS and other cloud provider is much easier. You can basically click a couple button on the Console and it will scale up or down in a second and when don't need it anymore, you can just terminate all the resource you made before and pay for only what you use.

Now lets talk about the feature inside EC2 :

## Instances
An Amazon EC2 instance is a virtual server. You will have full control over it from the moment you launch the instance until you terminate it (in AWS term, we called Launching an instance instead of starting an instance and Terminating an Instance instead of deleting an instance).
